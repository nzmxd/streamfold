# Streamfold X adapter

归页内置的 X 平台适配器。它作为可信签名平台能力随应用默认启用，也可由用户随时关闭；适配器通过受限浏览器会话被动读取 X 已登录网页自身产生的 JSON 响应，用于同步本人公开资料和本人原创或引用帖子。

## 数据范围

- 身份核验始终先等待 `account/settings.json` 的首个声明响应并读取登录 handle，再用 `UserByScreenName` 返回当前实际登录账号和稳定账号 ID；确认与同步前后都重新执行同一条会话绑定链，并由宿主核对预期稳定 ID。
- 支持仅资料、最近 20 条和最近 100 条三种范围。
- `recent_20` 最多监听 5 个时间线响应，`recent_100` 最多监听 20 个；若有界监听因响应数、时间或总量限制停止后仍有 Bottom cursor，会返回已验证的本人内容，并附带实际响应数、上限和内容数警告。
- 支持配置手动采集间隔，并按账号或分组创建自动运行计划。
- 排除回复、纯转帖及其他账号内容；帖子链接统一为 `https://x.com/i/web/status/{id}`。
- 映射浏览、点赞、回复、转帖、书签和引用数；不把账号主动点赞数当作收到的累计点赞。

## 安全边界

插件不使用 X 开发者 API、API Key、Cookie 导入、请求头读取、DOM 解析或自动验证码。登录和登录挑战必须由用户在 X 页面中手动完成。重复游标、冲突重复、响应结构错误、X 内部网页接口变化、限流或挑战仍会使同步失败。
