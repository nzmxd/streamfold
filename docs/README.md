# 设计文档索引

## 阅读顺序

1. [调研与总体设计](research-and-design.md)
   - 明确产品边界和停止条件。
   - 了解 GitHub 非官方接口与各平台接入建议。
   - 查看 Electron、内置 Chromium、插件宿主和本地存储架构。
   - 查看账号、分组、内容、指标和同步任务的数据模型。

2. [界面与功能设计](interface-and-feature-design.md)
   - 查看工作台、账号、内容、数据、插件和设置六个模块。
   - 查看账号分组、备注、标签以及内置浏览器交互。
   - 查看添加账号、登录校验、同步、异常恢复和导出流程。
   - 根据 P0、P1、P2 范围安排实现顺序。

3. [可交互界面原型](../prototypes/social-account-manager.html)
   - 切换账号分组并搜索账号。
   - 查看账号总览、内置浏览器、内容数据和设置页签。
   - 修改本地别名、备注与分组，体验保存反馈。
   - 原型使用静态示例数据，不会连接真实平台或保存真实凭证。

4. [JSON/CSV 导入格式](import-format.md)
   - 查看当前可执行的内置文件插件所接受的字段和约束。
   - 使用仓库中的 JSON/CSV 示例验证本地数据闭环。

5. [MVP 实现状态](implementation-status.md)
   - 查看当前代码已完成的账号、内容、统计、插件、任务和浏览器安全能力。
   - 查看尚未接入的真实身份识别、浏览器采集与自动同步范围。
   - 查看构建、测试和运行时冒烟结果。

6. [加密备份与恢复](backup-and-restore.md)
   - 查看 `.svbackup` 的范围、加密参数、回滚和登录会话边界。

7. [小红书登录身份核验适配器](xiaohongshu-identity-adapter.md)
   - 查看固定脚本、隔离执行、操作步骤和停止条件。
