# 加密备份与恢复

## 备份范围

`.svbackup` 保存归页当前 SQLite 数据库，包括账号本地资料、分组、备注、标签、内容、指标快照、任务、插件状态和应用设置。

以下内容不在数据库备份中：

- Chromium Session Partition 中的平台 Cookie、缓存和站点存储；
- 用户未主动下载到归页的图片或视频；
- 应用安装文件和操作系统凭证。

因此备份可用于恢复本地统计，但不能迁移或绕过平台登录。恢复后所有账号都会暂停同步，并要求重新打开官方页面核验身份。

## 加密格式

- 文件格式有独立标识和版本号。
- 使用 AES-256-GCM 认证加密。
- 密钥通过 scrypt 派生，参数固定为 N=65536、r=8、p=1，并使用随机 16 字节 salt。
- 每个文件使用随机 12 字节 IV 和 16 字节认证标签。
- 元数据通过 AAD 绑定；密文另有 SHA-256 完整性校验。
- 密码只在本次 IPC 调用和加解密内存中使用，不写入数据库、设置或日志。

密码无法找回。建议使用至少 12 个字符的独立密码，并与备份文件分开保存。

## 恢复保护

恢复按以下顺序执行：

1. 用户在系统文件选择器中选择 `.svbackup` 并输入密码。
2. 验证外层格式、认证标签、密文校验、数据库摘要和 48 MB 数据库上限。
3. 在临时文件中执行 SQLite 版本、`integrity_check` 和外键检查。
4. 进入全局维护状态，等待现有 IPC 操作结束，关闭账号浏览器并使平台 API 身份预览令牌失效。
5. 将当前数据库保留为临时回滚文件，再替换数据库；插件初始化和恢复时间写入也在回滚边界内。
6. 新数据库打开、迁移或后置初始化失败时恢复原数据库；全部成功后才删除回滚文件。
7. 清理恢复前后账号的 Chromium Session Partition，所有账号连接状态改为等待核验，同步保持暂停。

错误密码、损坏文件、未来版本或非 SQLite 数据不会替换当前数据库。

## 与 JSON/CSV 导出的区别

JSON/CSV 用于分析和数据交换，不包含任务、插件状态、同步游标等全部运行信息，也不能直接恢复数据库。需要完整恢复时使用 `.svbackup`。
