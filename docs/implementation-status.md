# Social Vault 0.3.0 实现状态

> 更新日期：2026-07-13

## 1. 已完成

### 1.1 桌面外壳与账号浏览器

- Electron 43、Vue 3、TypeScript 和 electron-vite。
- 管理窗口通过 `app://shell` 加载，每账号浏览器工具栏通过 `app://browser` 加载。
- 每个账号使用独立 `persist:social:<uuid>` Session Partition 和独立 1180×820 浏览器窗口。
- 远程页面没有 preload 或 IPC；Node.js、webview、弹窗、下载、权限、证书绕过和 HTTP Basic Auth 默认关闭。
- 导航、子框架和重定向只允许清单中逐项列出的精确官方 HTTPS hostname，不使用主域通配；显示 URL 会移除查询参数和片段。
- 断开会话清除该账号 Partition 的全部 Chromium 站点存储但保留账号历史；再次打开官方入口会进入“等待登录”，不会直接标记为已登录；永久删除是单独操作。

### 1.2 账号与状态模型

- 支持四个平台的多个本人账号空间，以及别名、备注、标签、默认账号、自定义分组、排序、搜索筛选和批量管理。
- 支持分组名称/颜色编辑、上下移动、当前筛选多选、批量加入/移出分组和批量暂停/恢复；只有身份核验为 ready 且未禁用的账号可批量恢复同步。
- 连接状态、身份归属、同步开关和任务状态已拆分，避免把“打开官方域名”误写成“已经登录”。
- 断开会话、清空历史数据和永久删除本地账号是三个独立动作。
- 账号详情可查看浏览器会话、本地内容、同步策略和危险操作说明。

### 1.3 版本化 SQLite

- 使用 `PRAGMA user_version` 执行 v0 → v1 → v2 → v3 事务迁移，兼容上一版本数据库。
- 已落地账号/内容指标快照、内容、插件安装、导入批次、持久任务、同步游标和应用设置表。
- 内容按 `(account_id, remote_id)` 去重；快照按 `(content_id, captured_at)` 幂等；内容、导入批次与成功任务在同一事务内提交。
- 导入身份冲突整批回滚；任务状态更新使用预期状态 CAS，重启后遗留任务标记为 interrupted。
- SQLite 数据模型不包含密码、Cookie 或 Token 字段；导入器拒绝常见凭证字段，并且不保存原始导入文件或完整导入路径。

### 1.4 内置文件与身份插件

- `generic-file-import` 为默认启用的内置、只读、本人账号限定、无网络权限文件插件。
- JSON/CSV 支持 camelCase/snake_case、CSV 引号/逗号/换行、账号资料、内容和指标快照。
- 文件限制 10 MB / 5,000 条内容 / 5,000 个快照；严格 UTF-8、HTTPS、日期、指标和敏感字段校验。
- 原生文件选择器、5 分钟一次性预览令牌、账号绑定、本人确认和事务提交已落地。
- 重复导入不会重复内容或相同时间快照；完整示例位于 `examples/`。
- 小红书 `probe + whoami` managed-browser 插件可以由用户主动启用；固定版本和 SHA-256 的内建脚本在 isolated world 运行，审核源码当前只读取创作中心页面可见 DOM。isolated world 本身不是网络或凭证能力沙箱。
- 首次绑定要求 profile link 与显式 user ID 两类证据一致，再执行“预览 → 本人确认 → 再次核验 → 提交”；身份变化会设置 mismatch 并停止同步。入口强制账号互斥、60 秒间隔和验证码冷却。微博、抖音和知乎仍只作为规划清单展示。

### 1.5 六个真实功能页面

- 工作台：本地账号、内容、浏览、互动、提醒和最近任务。
- 账号：双栏管理、独立浏览器、内容摘要、分组备注和会话/数据分离操作。
- 内容：关键词、账号、平台、类型、日期筛选，内容详情、指标变化、快照历史和本地备注标签。
- 数据：7/30/90/365 天趋势、账号排行、内容类型分布和跨平台口径提示。
- 插件：manifest、能力、风险、版本、来源、启停、任务列表和三步导入向导。
- 设置：版本与存储统计、未来调试响应策略、JSON/CSV 结构化导出、完整数据库加密备份恢复、按账号清空本地历史。

### 1.6 IPC 与导出

- preload 只暴露固定业务方法，不暴露原始 `ipcRenderer`、文件路径或数据库对象。
- 所有管理 IPC 校验主窗口、主框架和 `app://shell` 来源；浏览器 IPC 另行校验工具栏窗口与绑定账号。
- renderer 不能提交任意文件路径，文件打开和导出位置都由原生对话框选择。
- CSV 导出会转义引号并中和电子表格公式前缀；JSON 导出排除 Session Partition。

### 1.7 加密备份恢复

- 完整 SQLite 备份使用 AES-256-GCM、scrypt（N=65536/r=8/p=1）、随机 salt/IV、AAD 元数据绑定和 SHA-256 完整性校验。
- 备份格式、Base64、JSON、文件大小和数据库 `integrity_check`/外键/版本均严格校验；错误密码或认证篡改不会触碰当前数据库。
- 数据库替换失败会回滚原文件；成功恢复后关闭账号浏览器、暂停所有同步，并要求重新打开官方页面核验身份。

## 2. 尚未实现

- 小红书 `profile/contents/content_metrics` 读取和每次数据同步前的再次身份复验；当前只实现手动 `probe + whoami`。
- 微博、抖音、知乎的 `probe/whoami/profile/contents/content_metrics` 适配器，以及逐平台 SSO/认证弹窗白名单和专用测试账号验收。
- 自动/定时同步调度、平台单并发、Retry-After 冷却和连续失败熔断；界面当前只保存未来同步策略。
- ZIP/XLSX 导入导出和媒体文件备份。
- 分组拖拽排序、批量导出和浏览器窗口位置恢复。
- Windows/macOS/Linux 安装包、签名、自动更新和 CI 发布流水线。

这些能力不会通过伪造 UA、导入外部 Cookie、修改指纹、代理池、验证码服务或自动化平台写操作来实现。
