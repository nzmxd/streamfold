# 归页 Streamfold 0.4.0 实现状态

> 更新日期：2026-07-13

## 1. 已完成

### 1.1 桌面外壳与账号浏览器

- Electron 43、Vue 3、TypeScript 和 electron-vite。
- 管理窗口通过 `app://shell` 加载，账号浏览器工具栏通过 `app://browser` 加载。
- 每个账号使用独立 `persist:social:<uuid>` Session Partition 和独立浏览器窗口。
- 远程平台页面不加载业务 preload，不直接获得 IPC、文件系统或数据库能力。
- 导航、子框架和重定向按精确官方 HTTPS hostname 白名单校验。
- 断开会话只清理该账号 Chromium 站点存储；清空历史数据和删除本地账号是独立操作。

### 1.2 账号与状态模型

- 支持四个平台的多个本人账号空间，以及可选本地备注名、备注、标签、默认账号、自定义分组、排序、搜索筛选和批量管理。创建时备注名可留空，首次身份绑定后自动采用平台昵称；用户修改后不再被同步覆盖。
- 账号模型包含平台账号 ID、平台昵称、头像、简介和创作等级；账号详情读取最新快照展示关注、粉丝与累计获赞和收藏。
- 连接状态、身份归属、同步授权和任务状态分别存储。
- 只有当前会话身份经平台 API 核验、连接状态为 ready 且同步范围未禁用时，账号才能开启同步；用户主动同步时由后台自动取得账号 workspace lease，不以浏览器窗口是否可见作为前置条件。
- 初次身份核验使用 5 分钟有效的预览令牌；已有远端 ID 的账号可直接比较 API 身份。
- 同步前后均复验远端 ID，身份变化时整次任务不写入数据。

### 1.3 版本化 SQLite

- 使用 `PRAGMA user_version` 执行 v0 → v6 事务迁移。
- v4 使用数据库触发器约束同步授权；v5 增加累计获赞与收藏指标，并移除旧 DOM/文件插件安装状态和游标。
- v6 增加 `alias_customized`、头像缓存键与 MIME、简介和创作等级字段；关注、粉丝和累计获赞与收藏继续保存在 `account_snapshots` 时间快照中。
- 已落地账号、分组、账号快照、内容、内容快照、插件安装、持久任务、同步游标和应用设置。
- 内容按 `(account_id, remote_id)` 去重，快照按 `(content_id, captured_at)` 幂等。
- 资料、作品、指标快照和成功任务在同一事务内提交；失败任务保留错误状态。
- 旧版导入批次表只为数据库迁移兼容保留，当前没有文件解析服务、导入 IPC、导入插件或界面入口。

### 1.4 小红书 Session API 适配器

- 当前可执行插件为 `xiaohongshu-session-api`，默认关闭，由用户在插件中心启用。
- 基础身份与头像：固定请求 `GET /api/galaxy/user/info`，读取 `redId`、`userName`、`userAvatar` 和 `userDesc`。
- 账号资料与统计：固定请求 `GET /api/galaxy/creator/home/personal_info`，读取小红书号、昵称、简介、关注、粉丝、累计获赞与收藏和创作等级；其远端 ID 必须与 `user/info` 一致。
- 账号指标：在同一页面同源请求 `GET /api/galaxy/creator/data/note_detail_new`，保存 7 日与 30 日指标。
- 作品列表：打开作品管理页，通过 Chromium DevTools Protocol 捕获 `/api/galaxy/v2/creator/note/user/posted`，同时兼容已确认的旧路径 `/api/galaxy/creator/note/user/posted`。
- 作品管理响应的 `total`/`has_more` 显示仍缺页时，以有上限的 `PageDown` 输入触发页面继续加载；只读取随后产生的白名单 JSON 响应，不查询页面元素。
- 作品指标：打开数据分析页，捕获 `/api/galaxy/creator/datacenter/note/analyze/list` 的 XHR/Fetch JSON 响应，并按作品 ID 合并到作品列表。
- 传输层校验精确协议、主机、路径、HTTP 状态、JSON Content-Type、响应大小和字段结构。
- 支持 `profile_only`、`recent_20`、`recent_100`；作品最多保存 100 条，作品管理结果缺页或作品 ID 非法时拒绝提交。数据分析页未提供某项指标时保留为空值。
- 同账号操作互斥、同平台同步单并发，身份核验与同步均执行 60 秒最小间隔。
- 身份核验和主动同步通过后台隐藏 workspace lease 复用该账号的持久 Session；有效会话完成后自动释放后台 workspace，登录失效时才把同一官方浏览器窗口显示给用户。该生命周期与同平台单并发约束可直接复用于后续串行批量同步队列。
- 头像只接受小红书官方 CDN 的 HTTPS URL；下载过程校验重定向、图片 MIME、文件头、Content-Length 和 512 KiB 上限，按内容哈希写入本地缓存。Renderer 只访问同源 `app://shell/media/avatars/...`，不直接请求 CDN。
- 平台页面 DOM 不参与身份、资料、作品或指标采集，也不存在 DOM 回退路径。

微博、抖音和知乎插件以 `session_api` 清单展示为“计划中”，当前不能启用。

### 1.5 六个功能页面

- 工作台：账号、内容、浏览、互动和待处理状态摘要。
- 账号：双栏管理、平台头像与资料指标、独立浏览器入口、一键 API 身份核验与主动同步、内容摘要、分组和备注。
- 内容：关键词、账号、平台、类型和日期筛选，内容详情、指标变化和本地备注标签。
- 数据：7/30/90/365 天趋势、账号排行、内容类型分布和跨平台口径提示。
- 插件：平台插件清单、版本、能力、启停、最近运行时间和错误状态。
- 设置：版本与存储统计、JSON/CSV 导出、加密备份恢复和按账号清空本地历史。

### 1.6 IPC、导出与备份

- preload 只暴露固定业务方法，不暴露原始 `ipcRenderer`、文件路径或数据库对象。
- 管理 IPC 校验主窗口、主框架和 `app://shell` 来源；浏览器 IPC 校验工具栏窗口及绑定账号。
- 导出位置与备份文件由原生对话框选择。
- CSV 导出会转义引号并中和电子表格公式前缀；JSON 导出排除 Session Partition。
- 完整 SQLite 备份使用 AES-256-GCM、scrypt、随机 salt/IV、AAD 元数据绑定和 SHA-256 完整性校验。
- 恢复前先在临时文件执行版本、完整性和外键检查；成功恢复后关闭账号浏览器、暂停同步并重新核验身份。
