# Social Vault MVP 实现状态

> 版本：0.1.0
> 更新日期：2026-07-13

## 1. 本轮交付范围

已经形成可开发启动、生产构建和桌面运行的 Electron 应用，不再只是静态原型。

### 1.1 桌面应用

- Electron 43、Vue 3、TypeScript、electron-vite。
- 本地管理界面使用受限 preload 和显式业务级 IPC。
- 生产环境通过 `app://shell` 安全协议加载管理界面，不使用 `file://`。
- 主窗口采用账号列表与账号详情双栏布局，分组使用紧凑筛选器。
- 每个账号在独立的 1180×820 浏览器工作窗口中打开，避免窄栏嵌入和原生视图覆盖 Vue 弹窗。
- 管理界面包含账号、内容、数据、插件和设置入口；当前账号中心可用，其余入口明确标记为后续范围。

### 1.2 账号与本地管理

- 创建小红书、微博、抖音和知乎账号空间。
- 每个账号由主进程生成唯一 `persist:social:<uuid>` Session Partition。
- 支持本地别名、备注、标签、默认账号、同步范围和暂停状态。
- 支持自定义分组及一个账号加入多个分组。
- 删除分组不会删除账号或账号会话。
- 支持全部、未分组、登录异常、已暂停和自定义分组筛选，以及本地关键词搜索。

### 1.3 SQLite

- 使用 Electron 43 内置 Node.js 24 的 `node:sqlite`，无需第三方原生 SQLite 扩展。
- 自动创建账号、分组和账号分组关系表以及必要索引。
- SQLite 只记录 Session Partition 标识，不保存 Cookie、Token 或密码。
- 开启外键、WAL、超时和 defensive 模式。

### 1.4 内置 Chromium 安全层

- 每账号浏览器窗口由受限本地工具栏和全尺寸远程 `WebContentsView` 组成；远程页面没有 preload 或 IPC。
- 管理窗口与浏览器工具栏使用两个不同的最小 preload，浏览器工具栏只能操作其绑定账号窗口。
- 固定启用 `nodeIntegration: false`、`contextIsolation: true`、`sandbox: true` 和 `webSecurity: true`。
- 登录目标只能由主进程从内置平台清单选择，前端不能提交任意 URL 或 Partition。
- URL 必须为 HTTPS、无用户名密码、无异常端口，并匹配审核过的官方域名边界。
- 同时拦截普通导航、子框架导航和服务端重定向。
- 默认拒绝弹窗、下载、摄像头、麦克风、定位、通知、剪贴板和其他站点权限。
- 证书校验失败和 HTTP Basic Auth 均直接取消。
- 不修改 User-Agent、Canvas/WebGL、时区、字体或设备指纹。

### 1.5 IPC 边界

- preload 只暴露账号、分组和浏览器所需的固定方法，不暴露原始 `ipcRenderer`。
- 每个管理 IPC handler 校验发送方窗口、主框架和 `app://shell` 来源；浏览器工具栏 IPC 另行校验 `app://browser` 与窗口绑定账号。
- 主进程校验平台、状态、文本长度、数组数量、颜色和账号 ID；浏览器尺寸完全由主进程管理。
- 远程平台页面无法访问 SQLite、文件系统、插件或其他账号会话。

## 2. 尚未实现

- 不执行 `whoami`，因此当前不能自动确认远端账号身份。
- 不接入 OpenCLI、F2 或任何非官方采集适配器。
- 不读取个人资料、内容或统计指标。
- 不执行发布、评论、点赞、关注、私信或其他平台写操作。
- 内容中心、数据分析和插件中心目前为安全占位页。
- 尚未制作 Windows/macOS/Linux 安装包和自动更新渠道。

账号空间和官方登录入口已经可用，但在 `whoami` 与插件审计完成前，不应把登录成功解释为“已完成平台接入”。
