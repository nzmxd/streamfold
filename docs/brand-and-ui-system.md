# 归页 Streamfold 品牌与界面系统

> 更新日期：2026-07-13

## 1. 品牌

- 中文名：归页
- 英文名：Streamfold
- 定位：把分散在不同平台的个人账号、内容与数据，清晰地归于一个本地工作台。
- 口号：账号归位，内容成册。
- 标志：多条内容流折叠汇入一张页面，使用代码内 SVG 绘制，在不同像素密度下保持清晰。

应用内部继续保留原有数据库文件名、Session Partition 和备份格式标识，确保升级后已有账号、登录状态和历史数据可继续使用。

## 2. 色彩与主题

界面使用语义化设计 Token，页面和组件不依赖具体主题颜色。

| Token | 浅色 | 深色 | 用途 |
| --- | --- | --- | --- |
| `--bg` | `#F6F7FB` | `#0D1016` | 应用背景 |
| `--surface` | `#FFFFFF` | `#141821` | 主表面 |
| `--surface-subtle` | `#F0F2F7` | `#1A202B` | 导航、列表和次级表面 |
| `--text` | `#171A24` | `#F5F7FB` | 主要文字 |
| `--text-secondary` | `#5C6475` | `#B8C0CE` | 说明文字 |
| `--border` | `#E0E4EC` | `#2A3342` | 分隔与边框 |
| `--brand` | `#5859DE` | `#A5A6FF` | 鸢尾紫品牌色 |
| `--accent` | `#0A8F83` | `#45C8B5` | 潮汐青辅助色 |

主题支持浅色、深色和跟随系统三种偏好。选择结果同时保存到本地设置和 SQLite 应用设置；主进程通过 Electron `nativeTheme` 同步原生文件窗口、主窗口和账号浏览器。系统主题发生变化时，所有本地界面会立即更新。

## 3. 原生标题栏

主窗口和账号浏览器使用 Electron Window Controls Overlay：

- Windows / Linux 使用隐藏标题文本的原生覆盖层，保留系统最小化、最大化、关闭、Snap Layout 和辅助功能。
- macOS 使用 `hiddenInset` 并保留系统交通灯。
- 渲染层只绘制品牌、当前位置和主题入口；交互控件声明为不可拖拽区域。
- 标题栏颜色由主进程根据已解析主题设置，渲染层不能传入任意颜色或窗口方法。

## 4. 应用结构

- 顶部 48px：品牌、当前位置、外观切换和系统窗口控制。
- 左侧 214px：工作台、账号、内容、数据、插件和设置；每项同时提供名称与用途。
- 主区域：统一 24px 页面间距、16px 主容器圆角和语义化表面层级。
- 账号和内容页面继续使用列表与详情并列结构，避免频繁页面跳转。

## 5. 排版与可读性

界面采用语义化字号与行高 Token，不通过 Electron 全局缩放放大应用，也不改变平台网页的显示比例。

| 层级 | 字号 / 行高 | 用途 |
| --- | --- | --- |
| `--font-caption` | `12px / 18px` | 时间、角标和次要元数据；应用功能文字的最小字号 |
| `--font-secondary` | `13px / 20px` | 说明、状态与辅助信息 |
| `--font-body` | `14px / 21px` | 列表、表格与正文 |
| `--font-ui` | `15px / 22px` | 导航、按钮、输入框与页签 |
| `--font-section` | `16px / 24px` | 卡片和区块标题 |
| `--font-title` | `20px / 28px` | 详情与弹窗标题 |
| `--font-page` | `28px / 36px` | 页面标题 |
| `--font-metric` | `24px / 30px` | 主要数据指标 |

主窗口、应用弹窗与账号浏览器使用同一层级。账号浏览器只放大应用自己的工具栏文字，官方平台页面继续保持 Chromium 默认缩放，避免影响登录、验证码或平台页面布局。

## 6. 弹窗与反馈

应用自有确认操作统一使用 `AppDialogHost`，不再调用浏览器原生 `window.confirm`：

- 支持默认、提醒和危险三种语义。
- 打开后焦点进入取消按钮，Tab 在弹窗内循环，Escape 取消，关闭后焦点返回触发位置。
- 危险操作明确列出会删除和会保留的内容。
- 添加账号、创建分组等表单弹窗复用相同表面、阴影、按钮和焦点样式。
- 平台网页位于独立 `WebContentsView`，应用不替换平台自身弹窗。

## 7. 账号浏览器

账号浏览器是独立大窗口，不嵌在账号详情内。92px 顶部区域分为导航工具栏和状态栏：

- 显示当前账号名称与平台名称；名称按“本地备注名 → 平台昵称 → 平台账号”的顺序回退，添加账号时不强制填写备注名。
- 提供后退、前进、刷新和平台主页。
- 地址栏突出主机名、弱化路径，并显示官方域名状态。
- 页面加载时显示旋转标识和底部进度动画。
- 原生窗口控制不与页面工具栏重复；浏览器可独立切换浅色和深色外观。
- 远程平台内容从工具栏下方开始，保持完整可用高度。

主动核验与同步可以在不可见的后台 workspace 中复用相同 Session；只有登录失效时才显示这套完整浏览器界面。后台与可见窗口使用同一工具栏和主题状态，不创建第二份账号会话。

## 8. 实现边界

- 主窗口与账号浏览器只通过窄化 preload API 通信。
- 主题 IPC 只接受 `system`、`light`、`dark` 三个枚举值。
- 远程平台 `WebContentsView` 不加载 preload，不获得窗口或主题 IPC。
- 主题、标题栏和弹窗层不改动账号 Session、同步 API、数据模型或插件能力。
