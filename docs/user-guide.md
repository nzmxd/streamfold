# 使用指南

> 适用版本：归页 Streamfold 0.7.3

## 1. 当前可用平台

| 平台 | 本地账号与独立浏览器 | 身份/资料同步 | 本人内容与指标 |
|---|---|---|---|
| 小红书 | 可用 | 可用 | 可用，支持资料模式或最近 20/100 条 |
| 知乎 | 可用 | 可用，含官方周期指标 | 可用，支持资料模式或最近 20/100 条及动态指标 |
| X | 可用 | 可用 | 可用，支持资料模式或最近 20/100 条原创与引用帖 |
| 微博 | 可用 | 计划中 | 计划中 |
| 抖音 | 可用 | 计划中 | 计划中 |

小红书、知乎和 X 的数据同步插件默认关闭，需要先在“插件”页面启用。微博和抖音当前可以建立本地账号空间并使用独立浏览器，但同步按钮不会开放。

## 2. 添加并同步第一个账号

### 第一步：启用平台插件

进入“插件 → 已安装”，启用对应平台的数据同步插件，再为适配器确认账号与 Session JSON 权限。插件卡片会显示版本、贡献点、授权、计划和错误状态。

### 第二步：创建本地账号

进入“账号”，选择“添加账号”：

- 选择平台。
- “本地备注名”可以留空；首次身份绑定成功后会自动使用平台昵称。
- 选择同步范围：仅资料、最近 20 条、最近 100 条或不启用同步。

同一平台可以添加多个账号。每个本地账号都有独立 Session，不共享登录状态。

### 第三步：登录官方页面

打开账号详情的“浏览器”页签，选择“打开平台登录页面”。应用会打开独立大窗口，页面来自该平台的官方入口。按平台页面完成登录后可以直接关闭窗口，Session 会继续保留。

### 第四步：核验并绑定身份

返回账号详情，点击“核验当前账号”。首次核验会显示平台昵称和账号 ID；确认无误后选择“确认绑定此账号”。确认前应用会再次读取身份，避免核验期间发生账号切换。

确认令牌有效 5 分钟。过期或当前账号改变时，重新核验即可。

### 第五步：授权并同步

进入“设置与备注”：

1. 确认同步范围不是“不启用同步”。
2. 保存设置。
3. 选择“启用数据同步”。
4. 回到“总览”或“浏览器”，点击“立即同步”。

同步会自动在后台复用该账号 Session，不要求先打开浏览器窗口。只有登录失效时，应用才会显示同一个账号浏览器窗口供重新登录。

## 3. 账号整理

### 备注名、标签和备注

- 本地备注名只存在本机，不写回平台。
- 备注名留空时显示并跟随平台昵称；输入自定义名称后，后续同步不会覆盖。
- 标签用于账号搜索和整理。
- 账号备注可记录用途、负责人或登录说明。

### 分组

账号列表支持：

- 新建、重命名、改色、上移、下移和删除分组；
- 一个账号加入多个分组；
- 按未分组、异常、暂停或自定义分组筛选；
- 多选账号后批量加入/移出分组，或批量暂停/恢复同步。

删除分组不会删除其中的账号。

### 默认账号

每个平台可以设置一个默认账号。该标记用于本地管理，不会修改平台设置。

### 知乎创作指标

知乎账号完成同步后，账号详情会显示官方创作数据：

- 近 7 天、近 14 天、近 30 天和累计汇总；
- 最近 30 天每日趋势，可切换有数据的指标；
- 浏览、曝光、播放、赞同、喜欢、评论、收藏、分享、互动、发布量和完成率；
- 正向互动率与关注者转化，其中关注者转化可能为负数；
- 高级指标尚未达到平台阅读量或创作等级要求时，显示平台状态而不是 0。

日期范围按北京时间计算。平台后续修订同一周期数据时，归页会保留新的采集版本用于对比。

### X 内容口径

X 同步只保存当前绑定账号发表的原创帖和引用帖，不保存回复、纯转帖或时间线中出现的其他账号内容。通用指标分别表示浏览、点赞、回复、转帖和书签，引用数作为扩展指标保存。最近 20/100 条以符合该口径的本人内容为目标；页面在达到响应、时间或大小上限后仍表明存在未读取内容时，本次同步会失败且不会保存部分结果。

X 适配器复用独立账号浏览器中已经登录的网页会话，被动读取网页自身产生的只读 JSON 响应。它不需要开发者 API Key，也不会导入 Cookie、自动填写登录信息或处理验证码。

## 4. 批量同步与任务中心

### 创建同步批次

在账号列表中可以多选账号后选择“立即同步已选账号”，也可以在真实分组的操作栏直接选择“立即同步”。创建前可选择：

- 按每个账号保存的默认范围；
- 仅资料与账号指标；
- 最近 20 条内容；
- 最近 100 条内容。

临时范围只能等于或小于账号已保存的授权范围。应用会先预览实际账号，并检查登录、身份、同步开关和适配器；不能执行的账号会显示原因并跳过。可执行账号会分别创建任务，因此其中一个失败不会回滚其他账号已经完成的数据。

同一账号始终互斥，同一适配器按顺序执行，不同适配器可以并行。Session 有效时同步在后台完成，不会为了批量操作打开浏览器。

### 查看和处理任务

左侧“任务”页面提供：

- 排队、运行中、需要处理和今日完成摘要；
- 状态、处理状态、触发来源、平台、账号和时间筛选；
- 单任务阶段、进度、错误和尝试次数；
- 批次总数、完成数、失败数和剩余进度；
- 排队任务取消，以及失败/中断任务的新记录重试；
- 对确认无需继续处理的失败选择“标为已处理”；任务原记录和错误详情仍永久保留；
- 登录失效或身份异常时打开对应账号浏览器。

应用异常退出后，尚未开始的排队任务会在下次启动继续调度；已经进入核验或提交的任务会保留为“已中断”，由用户决定是否重试。原失败记录不会被覆盖。只有未解决失败计入“需要处理”；手动标记可将其消解，后续同一账号同步或同一插件贡献点成功也会自动消解对应旧失败。

托盘菜单会显示运行中和排队数量，可直接打开任务中心、暂停/恢复新的插件自动任务，并查看需要处理的任务数量。

## 5. 浏览器窗口

账号浏览器提供后退、前进、刷新、平台主页、当前官方主机状态和主题切换。平台网页保持 Chromium 默认缩放，应用主题不会修改平台页面样式。

关闭窗口只关闭视图；“退出登录”才会清除该账号的网页登录状态。重复打开同一账号时，应用会切换到已有窗口，不会创建第二份 Session。

## 6. 内容中心

同步成功后可以在“内容”页面：

- 全文搜索标题、摘要、标签和本地备注，并按账号、平台、分组、内容类型、标签、收藏、发布时间和采集时间组合筛选；
- 按相关度、发布时间、采集时间、浏览或互动分页排序；
- 查看标题、API 摘要、发布时间和最新指标；
- 比较最新与上一条指标快照；
- 查看快照历史；
- 为内容添加本地收藏、标签和复盘备注，或在当前页批量添加/移除标签及修改收藏状态；
- 按当前筛选范围导出 CSV，或导出可选包含完整快照的 JSON；
- 选择“浏览器中查看原帖”打开已校验的官方原帖链接。

平台没有提供的指标显示为空，而不是补成 0。只有指标发生变化时才会新增内容快照；即使数值没有变化，每次成功同步仍会保存一条只读观察记录，用于证明该内容在该次同步中确实被平台返回。

## 7. 工作台与数据分析

- “工作台”汇总账号数、内容数、浏览、互动、待处理状态和最近同步。
- “数据”提供概览、账号/平台/分组对比、内容生命周期和数据质量四个视图。
- 概览和对比只汇总语义一致的标准指标，展示当前值、绝对增量、增长率、样本数、缺失和数据修订。
- 生命周期在发布后 24 小时、7 天、30 天的容差窗口内选择最近观察；不插值，也不会用窗口外的快照估算。
- 数据质量展示账号最近同步和观察时间、未观察内容、缺少发布时间、缺失指标及最近一次成功同步警告。

插件未声明指标测量语义时按瞬时值展示，不自动累计增量。累计值回退会标记为“数据修订”，不会显示误导性的负增长；前值为 0 或字段缺失时增长率保持为空。

## 8. 数据导出、清理和备份

### 导出

设置页可以导出全部账号或单个账号：

- JSON：账号、内容、账号基础快照、账号周期指标历史和内容快照等结构化数据。
- CSV：适合在表格软件中查看的内容列表。

导出不包含 Chromium Session，也不能从文件反向导入平台数据。

### 三种清理操作

| 操作 | 保留内容 | 删除内容 |
|---|---|---|
| 清空历史数据 | 账号、登录、分组、备注 | 本地内容、指标、任务和同步游标 |
| 退出登录 | 账号资料、备注、内容和历史指标 | 该账号 Chromium Session |
| 永久删除 | 平台上的账号和内容不受影响 | 本机账号、Session、备注、内容、快照和任务 |

### 加密备份

`.svbackup` 保存完整 SQLite 工作区，密码至少 12 个字符。密码无法找回，应与备份文件分开保存。

备份包含账号、分组、备注、内容、指标、观察记录、任务、插件清单状态和应用设置，不包含网页登录状态、头像文件、插件包、插件 Secret、可重建的全文索引或外部导出文件。创建备份时会从便携数据库副本中清除全部加密 Secret，并停用第三方插件及其计划；恢复成功后应用会从内容表重建全文索引。文件使用 AES-256-GCM 加密，密钥由 scrypt 从密码和随机 salt 派生；版本、KDF 参数、随机 nonce、认证标签和密文摘要一同保存，用于恢复前校验。

恢复时应用会进入维护状态，等待当前操作结束并关闭账号浏览器，随后在临时位置完成格式、密码、schema、SQLite 完整性和外键检查。只有全部通过才替换当前数据库；失败会保留原数据。恢复完成后会清理账号 Session、暂停同步并要求重新登录和核验；第三方插件需要从签名目录重新安装，所有 Secret 需要重新填写。

JSON/CSV 用于查看或分析标准化数据，是明文且只有导出方向；`.svbackup` 用于恢复完整本地工作区，不能恢复平台登录。两者用途不同。

## 9. 主题、侧栏与更新

- 顶部外观入口支持浅色、深色和跟随系统。
- 左侧导航可以折叠，折叠偏好保存在本机界面存储中。
- Windows NSIS 和 Linux AppImage 等标签发布且明确启用更新源的正式安装包，默认自动检查公开 GitHub Release；发现新版本后在后台下载，下载完成才询问是否重启安装。
- 可在“设置 → 软件更新”关闭自动检查或手动检查。

开发版、目录构建、未签名 macOS 构件、Linux `tar.gz` 等不支持的运行形式会显示相应状态，不会尝试应用内安装。当前 macOS DMG/ZIP 需要从 Release 页面手动更新。

## 10. 常见状态处理

| 状态 | 处理方式 |
|---|---|
| 插件未启用 | 到“插件”启用对应平台适配器 |
| 尚未核验 | 登录后点击“核验当前账号”，首次使用需确认绑定 |
| 同步未启用 | 在“设置与备注”选择范围、保存并启用同步 |
| 登录已失效 | 在自动显示或手动打开的账号浏览器中重新登录，再核验 |
| 身份不匹配 | 检查浏览器里登录的账号是否与本地绑定一致；切回正确账号后重新核验 |
| 同步冷却 | 等待账号详情显示的冷却时间后再试，不要连续点击 |
| 没有内容摘要 | 平台详情 API 可能返回空摘要；小红书会在后续同步中继续补齐本地仍为空且未超出单次预算的项目 |
| 没有原帖入口 | 当前记录没有通过平台、主机、路径与远端 ID 一致性校验的官方链接 |
| 更新不可用 | 确认使用正式受支持的安装包；开发版和便携归档需要手动更新 |
| 批量同步跳过账号 | 按预览原因完成登录、身份核验、启用同步或缩小本次范围，再重新创建批次 |
| 任务已中断 | 应用上次退出时任务已经开始；确认账号状态后在“任务”创建新重试 |
| 失败不再需要处理 | 在任务详情选择“标为已处理”；记录不会删除，后续可按“已解决”筛选查看 |

## 11. 插件中心

- “已安装”管理插件和贡献点的启停、账号/分组/字段/网络授权、配置、Secret 和计划。
- “发现”从发布构建配置的签名目录安装或更新插件；目录未配置时不影响内置平台。
- “运行记录”查看手动、事件和定时任务，并可重试失败投递。
- “开发插件”启用开发者模式后选择本地 `.streamfold-plugin`；开发包会持续显示标识，且仍需逐贡献点授权。

定时贡献点创建计划时可以选择“按间隔”“每天”“每周”或“每月”。每天计划选择本地时间；每周计划选择一个或多个星期；每月计划选择一个或多个日期。计划按当前系统时区计算下次执行时间，应用关闭期间错过多个周期时只补执行一次；修改系统时区后，后续计算使用新的本地时区。所有新计划默认由用户明确选择范围和启用状态，连续失败三次仍会熔断暂停。

扩大权限、网络目标或数据范围需要重新确认。Secret 只允许写入或清除，不会在界面回显。卸载第三方插件会删除插件包、授权、配置和 Secret，但不会删除已经同步的账号、内容或指标。
